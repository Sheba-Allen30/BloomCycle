\chapter{DEPLOYMENT}
\section{Deployment Architecture}
Deploying a MERN stack application involves hosting both the frontend client and the backend server, and ensuring they communicate seamlessly over the internet.
\\\\
\textbf{Frontend Deployment}\\
The React frontend is built into static files using \texttt{npm run build}. These files can be hosted on platforms such as Vercel, Netlify, or GitHub Pages. These platforms provide continuous deployment capabilities, automatically rebuilding the site when new changes are pushed to the repository.
\\\\
\textbf{Backend Deployment}\\
The Express.js backend is deployed to a cloud hosting provider, such as Render or Heroku. The backend connects to the remote MongoDB database. Environment variables, such as the \texttt{MONGO\_URI} and \texttt{JWT\_SECRET}, are securely configured in the hosting environment.
\\\\
\textbf{Database Hosting}\\
The database is hosted on MongoDB Atlas, a fully managed cloud database service. Network access is configured to allow connections from the backend server's IP addresses.
