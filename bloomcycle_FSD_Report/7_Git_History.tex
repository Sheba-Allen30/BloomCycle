\chapter{GIT HISTORY }
To ensure proper version management and easy distribution, the entire Bloomcycle source code has been managed using Git. This allows for efficient tracking of updates, bug fixes, and feature improvements over time. By using Git, different versions of the web application can be accessed, making it easier to roll back changes if needed. The repository serves as a centralized location for storing and sharing the application codebase, ensuring that the latest production and development branches are always available for testing and deployment.
